# Diagnostics and Repair Standard

## Diagnostics

- Product version.
- OS and architecture.
- Installed runtimes.
- SQL components.
- Port status.
- Configuration validity.
- Paths and permissions.
- Database health.
- Sanitized recent errors.
- Integrity summary.

## Repair

- Back up before destructive actions.
- Repair only known-safe state automatically.
- Require confirmation for reset or restore.
- Record every action.
- Generate a support bundle when repair fails.
