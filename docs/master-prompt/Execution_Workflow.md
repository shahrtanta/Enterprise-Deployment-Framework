# Execution Workflow

1. Inventory.
2. Classify architecture.
3. Identify blockers.
4. Protect data.
5. Correct storage and configuration.
6. Correct first-run behavior.
7. Correct database bootstrap.
8. Correct HTTP/HTTPS behavior.
9. Localize offline assets.
10. Build publish pipeline.
11. Build launcher and tools.
12. Build installer.
13. Run build and tests.
14. Run publish validation.
15. Perform install and recovery tests where possible.
16. Generate final reports.
